===================================================================
ОПИСАНИЕ ПРОЕКТА И СТРУКТУРА
===================================================================
Консольная программа на языке C для генерации случайных паролей.

Особенности реализации:
* Поддерживает гибкие диапазоны длин: жестко заданная длина (-n) или случайный диапазон (-minl, -maxl).
* Позволяет генерировать заданное количество паролей (-c) за один вызов программы.
* Поддерживает гибкую настройку пула символов: через пользовательский алфавит (-a) или предопределенные классы символов (-C [aADS]).
* Реализует уникальный динамический автоматный парсер разделителей аргументов "на лету" (-d, -D).
* Осуществляет строгий контроль несовместимостей параметров (например, -n вместе с -minl/-maxl) и дубликатов с выводом ошибок в stderr.

Структура проекта:
* app/ — исполняемое приложение (точка входа, вызов инициализации, парсера аргументов и генерации).
* lib/ — библиотека функций (автоматный парсер разделителей командной строки и генератор паролей).

===================================================================
ПОДРОБНАЯ ИНСТРУКЦИЯ ПО СБОРКЕ И ЗАПУСКУ В POWERSHELL (WINDOWS)
===================================================================

1. Очистка кэша и сборка проекта

# Удаляем старую папку сборки (если она была), чтобы избежать конфликтов кэша:
> Remove-Item -Recurse -Force build

# Создаем чистую папку для сборки и переходим в нее:
> mkdir build
> cd build

# Настраиваем CMake:
> cmake ..

# Запускаем компиляцию проекта:
> cmake --build . --config Debug


2. Ручной запуск программы и тестирование опций

После успешной сборки исполняемый файл "my_program.exe" появится в подпапке 
"build/app/Debug/". Все команды ниже выполняются из папки "build".

* Пример 1. Стандартный запуск (генерация 1 пароля длиной 10 символов):
  > ./app/Debug/my_program.exe

* Пример 2. Генерация 5 паролей длиной 15 символов (класс только из цифр и спецсимволов):
  > ./app/Debug/my_program.exe -c:5 -n=15 -C:DS

* Пример 3. Слитное написание параметров (без пробелов) для диапазона длин от 8 до 12 символов:
  > ./app/Debug/my_program.exe -minl8 -maxl12 -c3 -C:SD

* Пример 4. Сложный тест с изменением списка разделителей "на лету" прямо в процессе разбора:
  > ./app/Debug/my_program.exe -d! -n!12 -D=- -c-3 -d= -C=A
